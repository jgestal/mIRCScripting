Las compraciones: if-then-else

      Las comparaciones (en otros sitios y en la ayuda del mIRC mencionados como if-then-else) son el centro de todo script, sin ellas no podríamos indicarle al mIRC que si tal persona dice tal palabra se le patee del canal, o si nos mencionan en un canal en el que no estamos muy atentos que nos avise con un echo... y un sinfín de cosas que entran dentro de las funciones "programables" del mIRC que la verdad son extensas a veces y que tambíen no suelen ser aprovechadas por las personas que usan el mIRC "a pelo" :-)

      Vayamos por partes, aquí tienes una tabla de todas las comparaciones que hay en el mIRC, algunas palabras y ejemplos aparecerán resaltados en negrita (los ejemplos irán además en cursiva. En los ejemplos de la tabla no voy a usar variables ni identificadores porque quiero que primero entiendas la función de cada operador. los ejemplos son alias

Operador Significado/función
==

igual a
Logicamente se refiere a que si cierto valor es igual a otro, generalmente se usan identificadores, variables o valores fijos. En cualquier comparación se pueden usar valores fijos o variables (para que luego no digan que no he mencionado eso)

igual if (blanco == blanco) echo -a Todo es blanco
===

igual a (sensible a las mayúsculas)
Hace casi lo mismo que el anterior, aunque no será lo mismo para éste Hola que hoLa, puesto que debe ser exactamente identico cada lado de la igualdad.

sensible {
    if (Ana === ana) echo -a Ana es lo mismo que ana
    else echo -a Um... parece que no es así
}

!=

distinto de
No creo que haga falta explicarlo

distinto if (casa != perro) echo -a parece ser que la casa no es un perro
<

menos que
Se usa para saber si un valor es menor que otro:

menor if (0 < 1) echo -a cero es menor que uno
>

más que
Se usa para saber si un valor es mayor que otro (nada nuevo...)

mayor if (1 > 0) echo -a uno es mayor que cero
>=

mayor o igual a
Por su propio significado cae no?

mayorogiual if (2 >= 2) echo -a se ha dado un valor mayor o igual a 2

mayoroigual2 if (5 >= 2) echo -a hay un valor mayor o igual a 2

menoroigual if (2 <= 3) echo -a el 2 es menor o igual que 3

isin

una palabra o letra está dentro de una cadena
Se usa para buscar coincidencias inexactas (ya que con esto especificas que cualquier palabra que contenga ese valor o cadena es válida

saludo if (a isin hola) echo -a parece ser que hay una a :-P

Debo apuntar que a partir del isin se utiliza una explamación para negar algo, por ejemplo:

saludo2 if (j !isin hola) echo -a no hay una "j" en "hola"

Así que en las estructuras que siguen tienes un funcionamiento parecido
isincs una palabra o letra está dentro de una cadena y además es sensible a las mayusculas
Parecido al isin, solo que influyen las mayúsculas
iswm

una cadena comodín coincide con otra cadena
Se usa para buscar coincidencias en un rango mayor que isin, mejor que veas el ejemplo para verlo

comodines if (m*l* iswm mola) echo -a parece que mola coincide
isnum

un valor es un número o pertenece a un rango númeríco
Extremadamente util en algunas ocasiones. Se usa para saber si una valor está dentro de un intervalo determinado (1-7 por ejemplo) o si ese valor es numérico o no, todo hay que decirlo y el segundo parámetro es opcional, mira el ejemplo:

numero if (4 isnum) echo -a el 4 es un número

numero2 if (2 isnum 1-5) echo -a el 2 es un número entre el 1 y el 5
isletter un valor es una letra o pertenece a una serie de letras
De funcionamiento parecido al isnum solo que con letras
ison

Tal nick está en tal canal
De uso realmente recomendado para saber si alguien está en un canal donde nosotros estamos (muy util en algunos casos la verdad)

100scripts if (Locorl ison #100scripts) echo -a Ahi tenemos en #100scripts al vIRCiado de Locorl
isop

El nick indicado es op en el canal indicado, util para acciones que tengan que ver con kicks o baneos la verdad

mirasitieneop if (Locorl isop #100scripts) echo -a Locorl tiene op en #100scripts
isvo

De funcionamiento parecido al isop, nos permite saber si un nick determinado es op o no en un canal

tienevoz if (SHaDoW isvo #100scripts) echo -a Ya tenemos otro bot inutil con voz en el canal
ishelp El helper (ayudante) sin mal no recuerdo es una especie de semi-op, aunque no se usa este en el irc-hispano (en ninguno de los 2) si creo recordar que se usa en Dalnet, no pongo ejemplo porque funciona EXACTAMENTE igual al isop e isvo
isreg Se refiere a si un nick es un nick "normal" (que no tiene op, voz, estatus de ayudante), paso de ejemplos por la razón dada en el ishelp
ischan

Interesante este... sirve para saber si el parámetro que le das es un canal donde estás, ejemplo ocupa una línea, pero por esas cosas de la vida la línea se parte :-P

escanal if (#100scripts ischan) echo -a De entre todos los canales en los que estás #100scripts es uno de ellos
isauto

Usado para saber si un determinado nick (o dirección, dependiendo de como lo tengas definido) está en tu lista de autoop de un canal.

esautoop if (Locorl isauto #100scripts) echo -a Locorl figura en la lista de autoops
isignore

Para saber si un nick está en nuestra lista de ignore, podemos indicarle que mire tambíen si tiene un tipo determinado de ignore.

esignore if (Locorl isignore) echo -a Locorl está en tu lista de ignore

esignore2 if (Locorl isignore t) echo -a Se están ignorando los ctcp's de locorl

[Aclaración] el t del isignore se refiere al parámetro -t del /ignore, con ello se ignoran los ctcp's de un determinado nick o de una determinada dirección
isprotect

Para saber si un nick está en la lista de nicks (o direcciones) protegidos del mIRC, funciona de igual forma que el isauto, al igual que este, el segundo parámetro es opcional

protegido if (Locorl isprotect) echo -a Locorl está protegido

prote if (locorl isprotect #100scripts) echo -a Locorl está protegido en #100scripts
isnotify

Ultimo ya! :-P. Este nos sirve para saber si un nick determinado está en el notify del mIRC

esnotify if (hPm isnotify) echo -a Warra King on the irc :-P

 

        Y como no, después de la explicación van los ejemplos, excepciones y explicaciones extensas, rollos increibles, blablablabla... Vamos a poner en práctica unos cuantos conceptos y enseñar un poco más de qué va esto. Probemos con un alias para saber el tamaño de un archivo, que llamaremos tamaño y nos devolverá el tamaño en bytes, kilobytes y megas:

tamaño {
    if (($lof($1-) >= 1024) && ($lof($1-) <= 1048575)) return $round($calc($lof($1-) / 1024),2) Kb
    if ($lof($1-) >= 1048576) return $round($calc($lof($1-) / 1048576),2) Mb
    else return $lof($1-) bytes
}

       Ahora expliquemos el ejemplo, vayamos por partes, si te fijas, el alias que estamos usando aquí está en modo de identificador (ya que si se cumple una condición u otra devuelve un valor), se han usado los identificadores $lof (tamaño del archivo), $round (redondeo a x décimas) y el $1- de parámetro, se usa este puesto que los alias que funcionan como identificadores toman las comas como separadores de parámetros (esto deberías sabertelo ya).
       No verías nada nuevo en esto si no fuese porque el primer if incluye una novedad, los &&. En los if puedes meter varias comparaciones a fin de que se cumplan todas (&&) o alguna de esas (||) o una combinacion de ambas para que si se cumplen una serie de condiciones o además se cumple otra se realice tal o cual acción.
       [Explicación del ejemplo] En el alias el primer if dice al mIRC que si el tamaño es mayor o igual a 1024 bytes y menor o igual a 1048575 bytes que devuelva el tamaño en kilobytes, en el segundo dice que si el tamaño es mayor o igual a 1048576 bytes el mIRC ha de devolver el tamaño en megas y por ultimo, el else quiere decir que si no se cumple ninguna de estas condiciones devuelva el tamaño en bytes¿lo entiendes?¿no? Pues pasate por el canal y pregunta por donde te quedas atascado :-). Eesto es un ejemplo, no han de ser estrictamente las mismas cosas :-P
