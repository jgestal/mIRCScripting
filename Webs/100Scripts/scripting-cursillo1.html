Aliases

Se encuentra en la opción Tools/Aliases o también en el icono verde /a

En esta sección puedes definir macros de los comandos tales como join, part, quit, opeando y desopenando y todo lo que quieras. También puedes poner cosas que dices normalmente... todo en general.

También puedes definir en ellos las teclas de función, o sea, desde F1 hasta F10, también con las combinaciones de shift + F? y con ctrl+ F?

En mIRC puedes usar o llamar un alias desde un menú del popup y desde el remote.

 

Creando Aliases

Empecemos con un pequeño ejemplo. Para meterte en un canal, normalmente escribes:

/join #canal

...donde #canal es el nombre del canal. Este es uno de los comandos que más se usa y lo conveniente es que se escriba o lo mas rápido posible o lo mas corto si puede ser, entonces creamos un aliases para esto /join #canal, o sea una macro: En vez de escribir /join #canal, hagamos algo que sea /j #canal con los mismos resultados, en este caso el aliases será:

/j /join $1

(De momento no te extrañes por lo que pone y no entiendas, es normal, estas empezando, así que no te preocupes, que lo explicare mas adelante)

Para probar si funciona debes poner /j /join $1 en los Aliases (botón verde con /a) y le das a aceptar, ahora si en la ventana del status pones:

/j #prueba

Entraras en el canal llamado prueba.

 

Sintaxis de los Aliases

Los Aliases se usan con este formato: /nombre_del_alias /comando.

En otras palabras, usando el ejemplo anterior, creamos una macro llamada /j luego un espacio y después el comando que queríamos que era /join $1.

Esto no termina aquí, aun queda mucho por explicar. Los Aliases también usan unas abreviaciones determinadas que son (#, $me y $! ) que siempre, y digo siempre, significan lo mismo para el mIRC, aunque hay algunas nuevas como $read [-l#] y $url que se explicaran mas adelante y algunas que son $1, *1, $$1, $?, #$1, #$$1, y #$? que también se explicarán...

No te preocupes, lo explicaré a su debido tiempo... jejeje.

Ahora nos fijaremos en $1. Después de todo es el que usamos en nuestro ejemplo de:

/j /join $1

$1 significa la primera palabra escrita después del comando del alias, así si escribes /j #prueba... $1 será #prueba, si escribes /j #ayuda... $1 será #ayuda.

 

Ejemplo: Múltiples Comandos

Ahora queremos intentar unirnos a distintos canales a la vez. Podemos crear un alias para hacer esto, solamente necesitamos separar los comandos con el carácter | que se consigue pulsando la tecla altGr + 1.

Para probar podemos usar como nombre de alias /j3 y continuarlo con los nombres de tres canales, para que nos unamos a la vez. El alias seria:

/j3 /join $1 | /join $2 | /join $3

Ahora pondremos eso en los aliases y probaremos usando lo siguiente:

/j3 #prueba1 #prueba2 #prueba3

...mIRC hará /join #prueba1, luego /join #prueba2, y luego /join #prueba3.

En este caso $1 es la primera palabra escrita, $2 la segunda y $3 la tercera.

Ahora, mIRC espera de ese alias 3 palabras, o nombres de canales. ¿Qué pasaría si pusiéramos sólo uno? por ejemplo /j #prueba1. mIRC daría un pequeño error pidiendo mas datos o que falta algo, para eso existe la opción $$1, que se explicara mas tarde.

Ejemplo: /say

Imaginemos que quieres decir algo al canal, como tu pagina Web o algo interesante, para eso usamos el comando /say comando. Un /say con mensaje manda el texto que le pongamos al canal o a una ventana de un privado (query)

/hola /say Hola a Todos...
/www /say La pagina de mIRC es http://metro.turnpike.net/mookies/mirc.html
/lineas /say Esta es la primera línea | /say Esta la 2ª | /say y esta la última línea

El ultimo ejemplo del /say te enseña a usar múltiples lineas de texto en un alias.

Sólo separa cada comando con un |. Como puedes ver, no todos los aliases necesitan las abreviaciones predeterminadas.

Ejemplo: /me

Acciones en los alias...

/d /me Piensa que esto es divertido
pega /me pega a $1 fuertemente

La segunda línea usa una abreviación $1. Si ponemos /pega María equivale a esto /me pega a María fuertemente

Ejemplos: Abreviaciones Compartidas

Múltiples comandos pueden compartir la misma abreviación:

/jt /join $1 | /topic $1 Solo probando...

En este alias el parámetro $1 es usado dos veces. Si ponemos /jt #prueba Primero entraríamos en el canal y luego cambiaríamos el comentario o topic de ese cana a Solo probando...

Otro ejemplo:

/amor /say ¡¡ Quiero a $1 ! | /say ¡¡ Realmente quiero a $1 !!

Sí pusiéramos /amor María pondría /say ¡¡ Quiero a María !! | /say ¡¡ Realmente quiero a María !!

Ejemplos: Cambiando Modos (Opeando, etc.)

Antes de poner nada tienes que saber el funcionamiento de los modos, para eso esta el manual del mirc, así que si no sabes como funcionan te remito a él para que te enteres.

/mode #canal +ooo nick1 nick2 nick3

...lo cual opeara a tres personas al mismo tiempo

Un bonito alias podría ser

/op /mode # +ooo $1 $2 $3

Para opear a Andrés, Paco y María pondrías /op Andrés paco María

... el símbolo # en este alias nos muestra el uso de otra abreviación predeterminada (# y $!). # siempre nos representa el nombre del canal del que usamos sobre el alias

En el mismo sentido podemos usar:

/deop /mode # -ooo $1 $2 $3
/ban /mode # +b $1
/kick /kick # $1
/dbk /mode # -o $1 | /kick # $1 | /mode # +b $1

IMPORTANTE: El espacio

Es muy importante que dejes una separación o espacio entre la abreviación predeterminada y el texto, si no lo haces así NO FUNCIONARA BIEN EL ALIAS.

/gusta /say ¡¡¡ me gusta $1!!!

...NO FUNCIONARA debido a que le falta un espacio entre la abreviación $1 y !!!. OJO CON ESTE FALLO QUE ES MUY FRECUENTE.

/gusta /say ¡¡¡ me gusta $1 !!!

...este si funcionará bien.

 

¿Dónde podemos usar los Aliases?

Los Aliases se pueden usar en las ventanas donde normalmente ponemos lo que pensamos o escribimos, también en los popups y en los remotes cosa que explicaré mas adelante. Las teclas que nunca usamos, o utilizamos pocas veces son las teclas de función, las cuales se pueden configurar en los aliases usando a nicks seleccionados o en el canal, dependiendo de como hayamos definido el alias. También podemos llamar a otros alias desde un alias o usarlos como un loop (bucle), para conseguir distintas cosas. Para que no se caiga en la tentación de cometer un fallo y crear un loop infinito (cosa que nos bloquearía el ordenador) existe un comando que es el /maxdepth que limita las veces que se repite algo, está puesto por defecto en 5 como máximo y en 1 como mínimo. Para lo que sirve este comando es ni mas ni menos que el alias pueda llamar a solo 5 alias más, con un nivel máximo de 100 si usamos /maxdepth 100 UNA SOLUCION CUANDO ENTRE UN ALIAS EN UN LOOP, PULSAD CTRL+ Pausa para parar el loop.

 

Alias Especiales; Las teclas de función

Hay un tipo de alias especial, el de las teclas de función que puede expresarse en la siguiente formula:

/f? /comando

Dónde /F? es la tecla de función (desde f1 hasta f12) y el /comando ya sabemos lo que es.

Ejemplo:

/f1 /say Hola a Todos
/f2 /topic $chan Esto es un nuevo Topic

la variable $chan es la del nombre del canal, se consigue el mismo efecto con #

Ahora si escribes /f1 dirás Hola a Todos... pero si pulsamos F1 saldar automáticamente lo de Hola a Todos

Mas ejemplos:

/f5 /me te da las gracias por enseñarme
/f6 /join #prueba
/f9 /dcc send $1 c:\mirc\mirc32.exe

la tecla F9 mandara el archivo mirc32.exe a un nick que DEBE estar SELECCIONADO en la ventana de los nicks, sino, tendrás un error por falta de nick. La tecla F9 puedes usarla en el canal, query y chats Private.

También puedes usar las combinaciones de shift + f? y ctrl + f?

Ejemplos:

/f1 /say Use F1 para poner esto
/sf1 /say Use shift + F1 para poner esto
/cf1 /say Use ctrl + F1 para poner esto

 

Usando corchetes

Podemos usar corchetes en los aliases para controlar el orden de evaluación de los identificadores. Los corchetes usados en los popups y en los remotes también. Los identificadores se evalúan primero desde la izquierda hasta la derecha. Los corchetes no presentan nada del resultado o comando. Por ejemplo

/note /notice $1 " $+ [ $read [ -s [ $2 ] ] c:\text\note.txt  $+ "
/sayvar /say % [ $+ [ $1 ] ]

Algunas veces tendrás que usar corchetes para forzar la evalución de determinados identifiers en alias, remotes o popups, de todas formas de los popups y los aliases nos encargaremos un poco más tarde

Lista de Identificadores

Es el turno de conocer todos los identificadores de los ALIASES, los de los popups y remotes los veremos mas adelante. A partir de aquí prueba todos los ejemplos hasta que comprendas el funcionamiento y puedas usarlo por ti mismo, si no te resultará difícil seguir...

$1 El mas fácil de todos. Solo contiene una palabra, o sea, la primera palabra después del alias. Con $2 cogerá la segunda, así para $3 la tercera...etc. Por ejemplo:

/op /mode $1 +o $2

Escribe /op #prueba Andrés paco para hacer operadores a Andrés y paco en el canal #prueba.

/golpea /me golpea a $1 con un $2

Escribe /golpea Juan mazo para obtener /me golpea a Juan con un mazo

*1 Este contiene muchas palabras. Contiene la primera palabra y todo lo que viene después de esa primera palabra. Probemos...

/decir /say Me gustaría decirte algo - *1

Si ponemos /decir mIRC es realmente divertido, con este alias nos dará: Me gustaría decirte algo - mIRC es realmente divertido. *2 tendría la segunda palabra y el resto... etc.

Prueba...

/note /notice $1 *2

... y pon /note nick texto.

También sería igual de válido usar $1- ya que hace el mismo cometido, por ejemplo:

/saludos /say hola $1-

...y poniendo /saludos Locorl, hPm, defkon (por ejemplo claro :-) saldria algo con el mismo efecto

$$1 Este comando solo se ejecuta si se le da un parámetro, si no, ignora el comando. Por ejemplo:

/j /join $$1

Ahora escribe /j, verás que no sucede nada, ya que mIRC ignora el comando ya que se espera algo y como no se da, pasa del comando. Si le ponemos /j #as $$1 funciona como un $1.

$? Este es un comando interesante... en un alias, si no queremos poner un parámetro fijo, como un nick, canal, etc, podemos poner un $? para que nos pregunte por el parámetro. Por ejemplo:

/j /join $?

/golpea /me golpea a $? con dureza.

$?="texto" Es un comando complementario al de $?, mIRC nos enseñara un cuadro, como en el ejemplo anterior pero con una modificación visible, o sea, que podremos poner el texto que queramos que aparezca en dicha ventana. Por ejemplo:

/j /join $?="¿Canal a unirte? con #"

/golpea /me golpea a $?="¿A quien quieres golpear?" con dureza

$! Es un comando que funciona conjuntamente con $? y sirve para recuperar lo que hemos puesto en el comando $?. Por ejemplo:

/j join $?="¿Canal a unirte? con #" | /say me gusta el canal $!

/amigo /say $? es mi mejor amigo | /say $! es el mejor...

#$1, #$$1 y #$? Son versiones especiales de los comandos conocidos $1, $$1 y $? pero que son usados solo con canales. Por ejemplo:

/j /join #$1

Podrás ver que en vez de usar /j #prueba solo con poner /j prueba hace lo mismo.

/j /join #$?="¿Nombre del canal? (sin #)"

# Este identificador contiene el nombre del canal donde es ejecutado el alias. Es importante por que hay muchos comandos (como el /mode) que requieren un nombre de un canal. Por ejemplo:

/op /mode # +o $1

Si escribes /op Andrés opearas a Andrés en el canal donde ejecutaste el comando.

$me Este identificador contiene tu nick actual, y puedes usarlo en varias situaciones. Por ejemplo:

/bueno /me piensa que mi nick ( $me ) es un buen nick...

/dar /me te ofrece el mIRC. escribe /ctcp $me xdcc send #1 para obtenerlo

/notoi /me esta away ( *1 ) | /away *1 | /say los mensajes que mandéis a $me serán grabados.

/nick /say me gusta mas el nick $1 que $me | /say $1 intenta cambiar de nick con /nick nick

$read [-l#] {filename.txt} Este identificador puede ser usado con varios propósitos. Su función es la de coger una línea aleatoria de un fichero especifico y la pone dentro del alias. Esa línea puede tener texto, comandos... Puedes usar $!read con el mismo efecto de $read, pero con la excepción de que no se evalúa la primera vez que se pone. Esto es bueno para usarlo en un /timer para $read evalúe cada vez que se pone.

/ja /say $read c:\mirc\risas.txt

/kick /kick # $1 $read c:\mirc\kicks.txt

Tambien puedes indicarle al $read que lea únicamente una línea específcia

/insulto /say $read -l5 insultos.txt

$readini ficheroini sección ítem Parecido a $read pero lee cosas de los ficheros ini. Esta creado para usarlo con el comando /writeini comando.
$url Este comando sirve para mostrar el la dirección actual que tienes en el navegador ya sea el Netscape o el Explorer (eso si lo tienes abierto)...

/mostrar /say He encontrado una pagina Bestial, mirad... $url

Identificadores con propiedades

Algunos identificadores pueden usarse con algunos parámetros añadidos, lo que nos ayuda en recordar su sintaxis y en hacer los scripts mas fácilmente. La sintaxis común para todos es  $identificador(N|#|nick).propiedad.

$server(N) Nos da acceso a la lista de servidores que sale en File/Setup/IRC_Servers.
$server(N) para la dirección, .desc para la descripción, .port para el puerto, .group para el grupo.
$ial(mascara,N) Nos da acceso a la lista interna de Usuarios
$ial(mascara,N) Para la dirección completa, .nick para el nick, .user para el usuario, .host para el host, .addr para la dirección.
$url(N) Nos da acceso a las direcciones URL para la dirección, .desc para la descripción, .group para el prefijo.
$chan(N/#).propiedad Nos devuelve todas las opciones del canal en el que estamos.
$chan(N/#) nos devuelve el nombre, .topic, .mode, .key, .limit,
$chat(N) nick, .ip, .status (activo, esperando, inactivo)
$fserv(N) nick, .ip, .status, .cd (directorio actual)
$get(N) nick, .ip, .status, .file, .size, .rcvd, .cps, .pc
$send(N) nick, .ip, .status, .file, .size, .sent, .lra, .cps, .pc
$timer(N) Devuelve el tiempo que has estado activo
$timer(N) para la id, .com, .time, .reps, .delay, .type

 

 

Identificadores definidos por el usuario

Después de todos los identificadores del mIRC, también puedes crear los tuyos que los puedes configurar y usar en todos los lados. Para crear un identificador es necesario que se creen en el aliases pero cin una excepción, que termine con el comando /return X. Por ejemplo, la suma de dos números:

/suma {
%temp = $1 + $2
return %temp
}

Ahora puedes usar el identificador $suma donde quieras, ya sea en aliases, remotes y popups.

Un ejemplo en un popup:

Suma:/say La suma de 45 y 78 es $suma(45,78)

O en el Alias:

/suma /say La suma de $1 y $2 es $suma( $1 , $2 )

OJO, SI USAS SOLO /SUMA NO FUNCIONARA.

Todos los parámetros que demos a los identificadores tienen que estar separados por comas o por espacios. También puedes usar estructuras if/then/else en las definiciones de los identificadores. Por ejemplo, un identificador que no da el tanto por ciento:

/porc {
if ($1 == $null) || ($2 == $null) { return No se han dado los datos suficientes }
if ($2 != 1) { %half = $2 / 2 } | else { %half = 1 }
%dummy = $1 * 100
%perc = %dummy / $2
%remainder = %dummy % $2
if (%perc == 0) { goto return }
elseif (%remainder != %half) { inc %perc }
:return
return %perc
}

La primera línea se asegura de que están todos los parámetros. El resto es el calculo del porcentaje cuya primera parte esta usada para ver si el total suma el 100%. No solo se pueden mandar números, también texto o mezcla entre ellos. Como ejemplo vemos como poner la hora en formato americano (7:52 pm):

/atime {
  set %hr $token(1,58,$time)
  set %min $token(2,58,$time)
  if (%hr == 0) { set %hr 12 | set %sub am }
  elseif (%hr < 12) { set %sub am }
  elseif (%hr == 12) { set %sub pm }
  else { %hr = %hr - 12 | set %sub pm }
  return %hr $+ : $+ %min $+ %sub
}

Con un poco de practica y dedicación podrás crear tus propios identificadores para satisfacer todo lo que tu quieras.
